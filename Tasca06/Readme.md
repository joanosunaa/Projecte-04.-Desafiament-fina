## T06: Accés Remot – Escriptori Remot (RDP) (Tasca individual)

## Breu descripció

En la tasca anterior heu après a administrar servidors i sistemes remots mitjançant SSH, l’eina essencial per a la gestió tècnica via terminal.
Però, què passa quan treballem amb Windows Server, amb entorn gràfic?
O quan un client ens diu:

“No em funciona el programa X.”

“Em surt un error a la pantalla.”

“No puc obrir aquest fitxer.”

Com a consultors d’EverPia, no només gestionem servidors; també oferim suport a usuari final, i això sovint requereix veure exactament el que ells veuen i poder utilitzar el ratolí i el teclat a distància.

Per això necessitem dominar l’Escriptori Remot.

## La Vostra Missió: Guies de Suport Gràfic (PoC)

En aquesta tasca, heu de crear la documentació que rebran els futurs becaris del departament.
Aquesta documentació explicarà, de forma clara i operativa, com utilitzar RDP per connectar-se a un equip amb escriptori gràfic.

Creareu una Prova de Concepte (PoC) utilitzant màquines virtuals.
L’objectiu és produir una guia fàcil d’entendre, precisa i ràpida de consultar, per permetre que qualsevol tècnic pugui establir connexions d’escriptori remot sense dificultats.

## Tecnologia a utilitzar
RDP (Remote Desktop Protocol)

L’estàndard de Microsoft per accedir a equips Windows amb escriptori gràfic.
Actualment, també disponible en entorns GNU/Linux, incloent Zorin OS 17/18, ja que GNOME incorpora suport natiu per RDP.

Això significa que ens podem connectar a:

Windows → Windows

Linux → Windows

Windows → Linux (Zorin)

Linux → Linux (Zorin amb GNOME RDP)

D’aquesta manera, el protocol RDP es converteix en una eina universal per al suport gràfic de l’empresa.

## Què ha d’incloure la guia?

A partir de la pràctica UD4.AA3 – Escriptoris Remots, la documentació ha d’explicar:

✔️ Com activar i configurar RDP al servidor o equip destí

✔️ Requisits de xarxa i seguretat

✔️ Com connectar-se des de Windows (Client RDP natiu)

✔️ Com connectar-se des de Linux (Remmina, GNOME Connections, etc.)

✔️ Validació de connexió, proves i captures de pantalla

✔️ Bones pràctiques per donar suport a clients

✔️ Solució d’errors comuns (ports bloquejats, credencials, permisos, firewall...)

Recorda: aquesta documentació l’utilitzarà un tècnic júnior sota pressió amb un client esperant.
Ha de ser clara, ràpida i intuïtiva.
